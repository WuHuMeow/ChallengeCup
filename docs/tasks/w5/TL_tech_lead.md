# W5 任务书：Tech Lead（TL）

> 周期：8/17（周日）- 8/23（周六）
> 核心目标：交付物质量把控、全员协调、确保报告和 PPT 初稿完成

---

## 每日任务

### Day 1（8/17 周日）
1. Review DA 的报告全文初稿：
   - 逻辑是否连贯
   - 数据是否准确（与 EX 的数据核对）
   - 是否覆盖 PDF 所有要求
2. 提出修改意见，跟踪修复

### Day 2（8/18 周一）
1. Review PPT 初稿：
   - 19 页是否齐全
   - 每页信息量是否合适（不过载不过空）
   - 与报告内容是否一致
2. 与 DA 协调修改

### Day 3（8/19 周二）
1. 检查 DB 的视频录制进度：
   - 素材是否齐全（路口 16 对比、高压力对比、云-边-端动画）
   - 旁白文字稿是否就绪
2. 如果素材不足，协调补录

### Day 4（8/20 周三）
1. Review 演示方案文档（DA 产出）：
   - 是否覆盖 PDF 要求的"场景描述、算法配置、演示脚本/流程、关键演示材料"
   - 路口 16 的描述是否准确
2. 提出修改意见

### Day 5（8/21 周四）
1. 全员进度检查：
   - DA：报告定稿了吗？PPT 定稿了吗？
   - DB：视频剪辑到什么阶段了？
   - 其他组：有无遗留 bug 需要修复？
2. 确定 W6 的最终打磨计划

### Day 6（8/22 周五）
1. 打 tag：`git tag v0.5-w5-complete`
2. 编写 W5 周报
3. 确认所有交付物初稿完成：
   - 报告 ✓
   - PPT ✓
   - 视频初剪 ✓
   - 演示方案 ✓
   - 代码仓库 ✓
   - 部署文档 ✓

### Day 7（8/23 周六）
1. Buffer：处理遗留问题
2. 准备 W6 的最终 review 清单
3. 确认提交方式（比赛平台上传？邮件？Git 仓库链接？）
