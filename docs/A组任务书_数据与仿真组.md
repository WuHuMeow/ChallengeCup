### 任务4：SUMO输出数据字段说明

整理仿真运行后产生的三类输出文件的字段含义：

**stats.xml（summary-output，每仿真步一行）：**

| 字段 | 含义 | 单位 |
|------|------|------|
| time | 仿真时刻 | 秒 |
| loaded | 本步加载的车辆数 | 辆 |
| inserted | 本步插入路网的车辆数 | 辆 |
| running | 当前路网中行驶的车辆数 | 辆 |
| waiting | 等待插入的车辆数 | 辆 |
| ended | 本步结束行程的车辆数 | 辆 |
| arrived | 本步到达目的地的车辆数 | 辆 |
| collisions | 累计碰撞次数 | 次 |
| teleports | 累计传送次数（车辆被强制移动） | 次 |
| halting | 当前停车（速度<0.1m/s超过一定时间）的车辆数 | 辆 |
| stopped | 当前停靠的车辆数 | 辆 |
| meanWaitingTime | 平均等待时间 | 秒 |
| meanTravelTime | 平均行程时间（-1表示无车辆到达） | 秒 |
| meanSpeed | 路网中所有车辆的平均速度 | m/s |
| meanSpeedRelative | 平均速度占限速的比例 | 无量纲 |
| discarded | 被丢弃的车辆数 | 辆 |
| duration | 本步计算耗时 | 毫秒 |

**traj.xml（fcd-output，每仿真步每辆车一行）：**

| 字段 | 含义 | 单位 |
|------|------|------|
| time（timestep属性） | 仿真时刻 | 秒 |
| id | 车辆ID（如 EW_car.0） | - |
| x | 车辆x坐标 | 米 |
| y | 车辆y坐标 | 米 |
| angle | 车辆行驶方向角 | 度 |
| type | 车辆类型 | - |
| speed | 当前速度 | m/s |
| pos | 在当前车道上的位置 | 米 |
| lane | 当前所在车道ID | - |
| slope | 道路坡度 | 度 |

**tripinfo-output（在.sumocfg中配置，车辆完成行程后输出一行）：**
- 全部20个路口的 `.sumocfg` 中 tripinfo 输出路径均为 `tripinfo.xml`
- 注意：路口1的 `stats.xml`/`traj.xml` 头注释中残留有 `runs/smoke.xml` 字样，那是上次运行（SUMO 1.27.1）的临时配置，并非当前 sumocfg 的设置，项目中也不存在 `runs/` 目录
- 典型字段包括：depart、arrival、duration、routeLength、timeLoss、waitSteps、departDelay等
- 具体字段需实际运行一次后从输出文件中确认

**queues.xml（queue-output，仅路口11-13、15-20有此配置）：**
- 这9个路口的 `.sumocfg` 额外配置了 `<queue-output value="queues.xml"/>`
- 记录各车道排队长度随时间的变化
- 典型字段包括：lane（车道ID）、time（时刻）、queueLength（排队长度）、queueLengthEnd（排队尾部位置）等
- 具体字段需查看已有的 queues.xml 文件或实际运行后确认
- 路口1-10、14无此输出配置

## 需要学习的知识

### 必学（开始工作前）

1. **SUMO基础操作**
   - SUMO是什么：开源微观交通仿真软件
   - 如何安装：https://sumo.dlr.de/docs/Installing/index.html
   - 命令行运行：`sumo -c xxx.sumocfg`（无GUI）和 `sumo-gui -c xxx.sumocfg`（有GUI）
   - 环境变量 `SUMO_HOME` 的配置方法（Windows系统）

2. **SUMO文件体系**
   - `.net.xml`：路网文件，定义节点(node)、边(edge)、车道(lane)、连接(connection)
   - `.rou.xml`：路由文件，定义车辆类型(vType)和车辆(vehicle)及其路径(route)
   - `.flow.xml`：流量文件，定义车流(flow)，指定起止边和车辆数
   - `.turn.xml`：转向比例文件，定义各进口道的转弯概率
   - `.sumocfg`：仿真配置文件，指定输入文件和输出文件
   - 推荐阅读：https://sumo.dlr.de/docs/Networks/SUMO_Road_Networks.html

3. **XML基础阅读能力**
   - 不需要会写XML，只需要能读懂属性和标签
   - 重点理解 edge、lane、connection、flow、vType 这几个标签的含义

### 选学（有余力时）

4. **SUMO输出文件格式**
   - Summary输出：https://sumo.dlr.de/docs/Simulation/Output/Summary.html
   - FCD输出：https://sumo.dlr.de/docs/Simulation/Output/FCDOutput.html
   - TripInfo输出：https://sumo.dlr.de/docs/Simulation/Output/TripInfo.html

5. **交通工程基础概念**
   - 什么是信号配时（周期、绿信比、相位）
   - 什么是转向比例（左转/直行/右转概率）
   - 什么是进口道、出口道、车道数

## 当前阶段不需要做什么

- **不修改任何代码**：不编辑 `algorithm.py`，不写Python脚本
- **不修改任何XML文件**：不调整路网、流量、配时
- **不运行算法**：不通过TraCI启动仿真（只用命令行 `sumo -c` 验证能否跑通）
- **不做对比实验**：不比较不同算法的效果
- **不写正式报告**：产出是共享知识库资料，不是最终技术报告
- **不做算法选型判断**：只整理数据特征，不建议"应该用什么算法"（那是B组的事）
- **不设计评价指标**：只说明输出文件有什么字段，不定义"应该用什么指标评价"（那是C组的事）
