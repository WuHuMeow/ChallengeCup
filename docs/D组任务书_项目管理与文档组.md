#### 4.2 PPT模板结构

| 页码 | 内容 | 时长建议 |
|------|------|---------|
| 1 | 封面（题目、团队、学校） | - |
| 2 | 目录/汇报大纲 | 30秒 |
| 3-4 | 背景与问题（窄路密网挑战） | 1分钟 |
| 5-6 | 方案总览（技术路线图） | 1分钟 |
| 7-9 | 仿真环境构建 | 1.5分钟 |
| 10-14 | 算法设计（核心原理+创新点） | 3分钟 |
| 15-18 | 实验结果（对比图表+数据） | 2分钟 |
| 19-20 | 演示视频关键帧/效果展示 | 1分钟 |
| 21-22 | 工程化与标准化价值 | 1分钟 |
| 23 | 总结 | 30秒 |
| 24 | 致谢/Q&A | - |

**PPT设计原则：**
- 每页一个核心信息，不堆砌文字
- 图表为主、文字为辅
- 关键数据用大字号突出
- 配色统一（建议2-3种主色）
- 总页数控制在20-25页

#### 4.3 演示视频脚本大纲

**总时长：5-8分钟**

| 时间段 | 内容 | 画面 | 解说要点 |
|--------|------|------|---------|
| 0:00-0:45 | 场景导入 | 雄安新区实景/规划图 + 路口地图 | 介绍窄路密网特征、交通管控挑战 |
| 0:45-1:30 | 问题展示 | SUMO仿真画面（固定配时，拥堵） | 展示现状问题：排队、延误 |
| 1:30-3:00 | 算法介绍 | 算法原理动画/流程图 | 核心思路、创新点（简明扼要） |
| 3:00-5:00 | 算法部署与运行 | SUMO-GUI仿真画面（算法控制） | 实时展示算法调控过程，数据指标叠加 |
| 5:00-6:30 | 效果对比 | 左右分屏/前后对比 + 数据图表 | 量化提升效果（延误降低X%、通行量提升X%） |
| 6:30-7:30 | 总结 | 关键数据汇总 + 应用场景展望 | 算法价值、推广前景 |
| 7:30-8:00 | 片尾 | 团队信息、致谢 | - |

**视频制作要点：**
- SUMO-GUI录屏需要高分辨率（1920×1080）
- 数据指标叠加：在画面角落实时显示当前排队长度、平均速度、累计通行量
- 效果对比：用相同视角、相同时间段的前后对比最有说服力
- 解说：语速适中，专业术语需简要解释
- 背景音乐：轻柔，不干扰解说

## 当前阶段不需要做什么

- **不编写任何代码**：不写Python、不写脚本、不配置SUMO
- **不做任何实验**：不运行仿真、不收集数据
- **不撰写正式报告正文**：只搭框架（目录结构），不写各章节内容
- **不制作正式PPT**：只确定页面结构和设计原则，不填充内容
- **不录制/剪辑视频**：只写脚本大纲，不实际制作
- **不做算法选型判断**：不评价"哪个算法好"（那是B组调研+团队讨论的事）
- **不定义评价指标**：不决定"用什么指标评价"（那是C组的事）
- **不修改原始数据**：不调整intersection_data中的任何文件
- **不催促其他组**：跟踪进度是"记录和同步"，不是"催交"
