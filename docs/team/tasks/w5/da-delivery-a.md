## 每日任务

### Day 1（8/17 周日）

- [ ] 根据 TL 的 review 意见修改报告全文
- [ ] 重点核对数据准确性：与 EX 确认所有百分比、p 值、指标数值无误
- [ ] 检查逻辑连贯性：章节间过渡自然，论证链完整
- [ ] 检查格式规范：图表编号、引用、页眉页脚、页码
- [ ] 目标：报告达到"可提交"质量

**验证：** ① TL review 意见逐条处理完毕；② 抽查 5 个数据点与 `output/csv/` 原始数据一致；③ 页眉显示项目名称、页脚显示页码；④ 目录页码与正文对应。

### Day 2（8/18 周一）

- [ ] 报告最终定稿：全文通读最后一遍
- [ ] 检查错别字、标点（Word 拼写检查 + 人工复核）
- [ ] 确认目录自动生成正确（更新域代码）
- [ ] 导出 PDF 版本（备用提交）
- [ ] 提交给 TL 做最终确认

**验证：** ① Word 拼写检查零警告；② 右键目录→更新域→页码正确；③ PDF 导出成功，页数合理（预计 25-35 页）；④ TL 确认"可提交"。

### Day 3（8/19 周二）

- [ ] PPT 定稿：根据 TL review 意见修改
- [ ] 确认 19 页内容完整，所有图表为最终版（DB 产出）
- [ ] 确认动画/转场简洁（仅淡入，无花哨效果）
- [ ] 排练计时：控制在 12-15 分钟
- [ ] 提交给 TL

PPT 定稿检查清单：

```
□ 第 1 页：封面信息完整（题目、团队、日期）
□ 第 2-3 页：背景数据准确
□ 第 4-5 页：架构图清晰
□ 第 6-8 页：公式正确、可读
□ 第 9-10 页：仿真环境截图清晰
□ 第 11-13 页：系统架构 + 稳定性数据
□ 第 14-16 页：实验数据为最终版
□ 第 17 页：工程化内容完整
□ 第 18 页：创新点 3+1 条
□ 第 19 页：团队分工 + 致谢
□ 全局：字体统一、配色一致、无错别字
□ 计时：12-15 分钟
```

**验证：** ① 19 页逐页检查清单全部打勾；② 排练计时在 12-15 分钟范围内；③ 无动画或仅有简单淡入；④ TL 确认"可提交"。

### Day 4（8/20 周三）

- [ ] 演示方案定稿：确认覆盖 PDF 所有要求

**验证：** ① 演示方案覆盖 PDF 中"实际场景应用演示"的全部要求；② 路口 16 数据与报告 4.4 节一致；③ Word 版格式与报告统一（字体、页边距）；④ TL 确认。

### Day 5（8/21 周四）

- [ ] 编写视频旁白最终稿（与 DB 协调）

- [ ] 确认数据引用准确（与报告第四章一致）
- [ ] 控制总时长 7 分钟（语速约 200 字/分钟，总计约 1400 字）
- [ ] 发给 DB 录制旁白
