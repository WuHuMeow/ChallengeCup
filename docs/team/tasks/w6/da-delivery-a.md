## 每日任务

### Day 1（8/24 周日）

- [ ] 参加全员 review 会议，记录报告/PPT 的修改意见
- [ ] 整理修改清单，按优先级排序（数据错误 > 逻辑问题 > 格式问题）
- [ ] 确认各项修改的负责人和截止时间
- [ ] 与 TL 确认答辩时间和评委关注点

**验证：** ① 产出修改清单文档（含优先级、负责人、截止日）；② 清单中每项有明确的完成标准；③ TL 确认清单无遗漏。

### Day 2（8/25 周一）

- [ ] 报告最终修改：修复 review 中发现的所有文字问题
- [ ] 确认图表编号连续、引用正确（全文最后一遍）
- [ ] 确认页眉页脚、页码、目录正确
- [ ] 最终导出 PDF（Word + PDF 双版本）
- [ ] PPT 最终修改：修复内容/排版问题，确认 19 页完整、动画简洁

**验证：** ① 报告 Word 和 PDF 内容一致、排版无错位；② PPT 19 页完整无缺页；③ 修改清单中"数据错误"和"逻辑问题"类全部关闭；④ 文件命名规范：`XH-202613_赛道B_团队名_报告.docx`。

### Day 3（8/26 周二）

- [ ] 演示方案最终修改（根据 review 意见）
- [ ] 确认所有文字交付物格式统一：字体、字号、行距一致
- [ ] 确认图表风格一致（配色、线型、标注方式）
- [ ] 确认引用格式一致（GB/T 7714）
- [ ] 提交给 TL 做最终确认

**验证：** ① 报告、演示方案字体/行距/页边距完全一致；② 所有图表配色统一（如 CA-MP 用蓝色、FixedTime 用灰色、Actuated 用橙色）；③ TL 确认"全部可提交"。

### Day 4（8/27 周三）

- [ ] 准备答辩主讲稿：按 PPT 19 页写逐页讲稿
- [ ] 控制总时长 12-15 分钟（每页约 40-50 秒）
- [ ] 标注每页的重点语句和过渡语
- [ ] 自己排练 2 遍，计时并记录超时/不足页面

答辩讲稿模板：

```
【第 1 页 封面】（10s）
"各位评委好，我们是 XX 团队，今天汇报的题目是..."

【第 2 页 问题背景上】（40s）
"雄安新区采用'窄路密网'规划理念，路段平均长度仅 XXm..."
→ 过渡："这样的路网给信号控制带来了什么挑战？"

【第 3 页 问题背景下】（40s）
"我们分析数据发现，最短边仅 24m，仅容 3-4 辆车排队..."
→ 过渡："针对这些问题，我们设计了 CA-MP 算法。"

...（逐页类推）

【第 19 页 致谢】（15s）
"以上就是我们的汇报，感谢各位评委，请批评指正。"
```

**验证：** ① 讲稿覆盖 19 页，每页有对应文字；② 排练 2 遍计时均在 12-15 分钟；③ 标注了重点和过渡语；④ 无数据错误（与 PPT 一致）。

### Day 5（8/28 周四）

- [ ] 整理提交材料：报告（Word + PDF）、PPT、演示方案
- [ ] 确认文件命名规范：`XH-202613_赛道B_团队名_报告.docx` 等
- [ ] 与 TL 确认比赛平台要求的格式和大小限制
- [ ] 将所有 DA 负责文件打包，发给 TL 统一提交

**验证：** ① 文件命名符合 `XH-202613_赛道B_团队名_XXX` 格式；② 文件大小在平台限制内；③ TL 确认收到全部 DA 文件；④ 打包文件可正常打开。

### Day 6（8/29 周五）

- [ ] 参加模拟答辩（担任主讲）
- [ ] 记录评委（TL/组员）反馈：语速、逻辑、重点突出度
- [ ] 根据反馈调整讲稿（重点修改超时或表达不清的页面）
- [ ] 再排练 1 遍，确认改进有效

**验证：** ① 模拟答辩完整走完 19 页 + Q&A 环节；② 计时在 12-15 分钟；③ 反馈问题已修改；④ 最后一遍排练流畅无卡顿。

### Day 7（8/30-8/31）

- [ ] 8/30：协助最终提交（确认平台上传成功、文件完整）
- [ ] 8/30：确认提交回执/截图保存
- [ ] 8/31：Buffer，处理任何最后问题
- [ ] 8/31：整理项目文档归档（确保所有版本可追溯）

**验证：** ① 比赛平台显示提交成功；② 7 项材料全部上传；③ 提交回执已截图保存；④ 项目文档归档完成。
