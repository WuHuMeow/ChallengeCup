# W3 任务书：Tech Lead（TL）

> 周期：8/3（周日）- 8/9（周六）
> 核心目标：确保全量实验顺利运行、协调各组产出、质量把控

---

## 每日任务

### Day 1（8/3 周日）
1. 确认 W2 预跑结果（路口 1-10 原始流量）无异常
2. 启动 W3 全量实验：路口 11-20 × 3 算法 × 原始流量 × 3 种子 = 90 组
3. 监控运行状态，处理失败实验

### Day 2（8/4 周一）
1. 检查实验进度：目标完成 180/360 组（原始流量部分）
2. Review DB 产出的图表质量：配色、标注、分辨率
3. Review DA 的报告初稿（第一/二/三章），提出修改意见

### Day 3（8/5 周二）
1. 继续监控实验运行
2. 抽查 3-5 个路口的实验结果：CA-MP 是否优于基线
3. 如果某路口 CA-MP 效果不佳，与 AB 分析原因（可能是参数不适配）

### Day 4（8/6 周三）
1. 原始流量 360 组实验应全部完成（含 W2 预跑的 90 组）
2. 用 collector.py 采集全部结果
3. 用 analysis.py 生成汇总对比表
4. 将完整数据发给 DA（报告）和 DB（图表）

### Day 5（8/7 周四）
1. Review 全部实验数据：
   - 20 路口中 CA-MP 优于 FixedTime 的比例（目标 > 80%）
   - 异常路口标记（CA-MP 反而更差的）
   - 与 AB 讨论异常原因
2. 确认数据质量：无缺失、无异常值

### Day 6（8/8 周五）
1. 打 tag：`git tag v0.3-w3-complete`
2. 编写 W3 周报
3. 确认 W4 计划：1.5 倍压力测试 + EWMA 接入 + Docker

### Day 7（8/9 周六）
1. Buffer：处理遗留问题
2. 开始准备 W4 的 Docker 环境（与 IA 协调）
